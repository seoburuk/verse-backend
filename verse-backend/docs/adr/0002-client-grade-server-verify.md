# ADR 0002: 클라 1차 채점 + 서버 재검증

## 상태
채택

## 맥락
앱이 본문을 먼저 보여주므로 정답이 이미 기기에 노출된다. "보안용 서버 채점"은
명분이 약하다. 그러나 진도/연속일 기록은 신뢰할 수 있어야 한다.

## 결정
채점은 클라에서 즉시(재미=즉각 피드백). 단, 기록을 바꾸는 제출은 서버가
자기 정답지로 재채점해 검증 후 저장.

## 근거
- 즉각 피드백은 재미의 핵심(기획서 §1).
- 서버 재검증으로 기록 무결성 확보(조작 방지).
- 모든 제출이 아니라 "결과 확정" 제출만 검증해 비용 절충(기획서 §3).

## 리스크
- 채점 규칙을 클라/서버 양쪽이 미러링 → 어긋나면 신뢰 붕괴.
