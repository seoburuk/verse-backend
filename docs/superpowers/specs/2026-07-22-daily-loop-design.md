# 데일리루프 설계 (2026-07-22)

PIXBIBLE(Flutter 앱)에 **매일 재방문 습관을 만드는 리텐션 엔진**을 도입한다.
사용자가 의미 있는 암송 목표를 직접 정하면, 앱이 그것을 오늘치 목표로 쪼개주고,
매일 채우면 픽셀 펫이 반응하며 진행도가 눈에 띄게 차오른다.

- **대상 플랫폼**: 앱(Flutter) 전용. 백엔드는 필요한 만큼만, 오프라인 우선(로컬 drift) 유지.
- **범위 상태**: 방향 확정 · 세부는 추후 수정 예정(§9 열린 질문 참조).

---

## 1. 컨셉 한 줄

> 사용자가 "이번 달에 시편 23편 외우기" 같은 **암송 플랜**을 정하면, 앱이 **오늘치 목표**로
> 쪼개주고, 매일 그걸 채우면 **픽셀 펫이 기뻐하고 진행바가 차오른다**. 안 하면 펫이
> 배고파하며 저녁에 알려준다. 플랜을 다 외우면 **다른 장을 지정해 루프가 이어진다**.

핵심 통찰: 현재 스트릭은 "아무거나 1절"이라 **목적이 없다**. 사용자가 고른 목표(특정 장/N절)를
일일 목표로 자동 분해하면, 매일 앱을 여는 이유가 "내가 정한 그 장을 끝내기 위해"로 바뀐다 —
이것이 리텐션의 핵심 동력이다.

---

## 2. 접근 결정

- **A(사용자 지정 플랜 → 일일 목표 자동 산출)를 뼈대로, C(프리셋 챌린지)를 병행**한다.
- 보상은 **게임적 재화(카드/XP/빙결) 없음**. 진행도 갱신 + 펫 상호작용(칭찬/반응)이 곧 보상이다.
- **다마고치 회귀 결정 명시**: 과거 "다마고치→카드 수집 컨셉으로 확정"했던 방향을 되돌려
  **펫 결속 메커니즘으로 회귀**한다. 단, 역할을 분리한다 —
  - **카드 수집** = 섹션 완료 수집물(기존 유지).
  - **펫** = 데일리루프 전용 동반자(신규).
  - 두 시스템은 겹치지 않으므로 공존한다.

---

## 3. 펫 결속 메커니즘 (재투자 훅)

- 펫 상태 = **데일리 목표 달성 이력**으로만 구동한다(별도 재화 없음).
- 오늘 목표 완료 → 펫 "배부름/행복" 애니메이션 + 칭찬 말풍선.
- 오늘 미완료 → 시간이 지나며 펫이 단계적으로 배고파함(표정 단계).
- 기존 **스트릭 위험 알림(저녁)을 "펫이 배고파해요 🍞" 문구로 재활용**한다.
- **톤 주의**: 죄책감을 과하게 주지 않는다 — "굶어 죽음" 대신 "시무룩/배고픔" 수준까지만,
  복귀하면 즉시 회복. (듀오링고식 죄책감 밈의 역효과 방지)

---

## 4. 홈 "오늘" 화면 (앱 시작 화면)

앱 시작 위치를 현재 `/courses`에서 **새 "오늘" 홈**으로 변경한다. 코스 탐색은 홈에서 진입하는
하위 탭/화면으로 내린다.

레이아웃(위→아래):
1. 상단: 픽셀 펫(상태 반영) + 스트릭 불꽃 + 목숨.
2. 중앙: **오늘 목표 링/바** — "오늘 3절 중 1절" 식으로 크게, 눈에 띄게.
3. 그 아래: **플랜 전체 진행바** — "시편 23편 6/23절 (이번 달 D-18)".
4. 메인 CTA: **"이어서 외우기"** → 플랜의 다음 절로 바로 진입.
5. 하단: 코스 탐색 / 대시보드 / 랭킹 진입.

---

## 5. 플랜 모델 (A + C)

- **플랜 = 대상 + 기간**.
  - 대상: 특정 장 / 코스·섹션 / "N절".
  - 기간: 이번 주 / 이번 달 / 직접 날짜.
- **C 병행**: 플랜 생성 화면에서 프리셋 시작점 제공(예: "시편 23편 30일", "주기도문 1주").
  자유 설정과 프리셋을 한 화면에서 고를 수 있게 한다.
- **완주 시**: "🎉 시편 23편 완성!" → **다음 플랜(다른 장) 지정 화면**으로 유도해 루프를
  지속시킨다. (목표 달성 후 이탈 방지의 핵심)

---

## 6. 오늘치 목표 자동 산출

- `오늘 목표 = ceil(남은 절 ÷ 남은 일수)`.
- 뒤처지면 자동 재계산해 남은 기간 안에 따라잡도록 한다.
- **"오늘치 완료" 판정** = 오늘 목표 절 수만큼 green 달성. 완료 순간 스트릭 기록 + 펫 반응.
- **딕테이션 모드는 기존 규칙대로 진행도 미반영**(채점 계약 §7 유지).

---

## 7. 데이터 & 재사용

**신규 (로컬 drift 테이블):**
- `memorization_plan` — 대상(장/코스/절수) · 기간(시작·마감) · 진행 상태.
- `daily_progress` — 날짜별 달성 절 수(오늘치 완료 판정용).
- `pet_state` — 펫 표정/허기 단계.

**재사용:**
- `StreakRepository`(`lib/core/db/lives_streak_repository.dart`) — 오늘치 완료 시 `recordActivityToday()` 호출.
- 알림 3종(`lib/core/notifications/reminder_service.dart`) — 위험 알림 문구를 "펫 배고픔"으로 교체,
  나머지 로직 재사용.
- 카드 수집(섹션 완료) — 손대지 않고 분리 유지.

**원칙**: 오프라인 우선. 전부 로컬 drift로 먼저 동작하고, 서버 동기화는 나중 단계.

---

## 8. 범위 / 단계 (Phasing)

- **P1** — 플랜 생성 + 오늘 홈 + 진행바 + "이어서 외우기". 펫 없이도 완결 동작.
- **P2** — 펫 상태·애니메이션 + 알림 문구 재활용("펫이 배고파해요").
- **P3** — 프리셋 챌린지 + 완주→다음 플랜 유도 + 서버 동기화.

---

## 9. 열린 질문 (추후 수정 예정)

사용자가 "나중에 더 다듬겠다"고 명시한 항목들. 구현 계획 수립 전 확정 필요.

- 펫의 구체 표정 단계 수와 허기 진행 속도(몇 시간/며칠 단위?).
- 펫 회복 규칙(오늘치 완료만? 부분 진행도 반영?).
- 플랜 "기간" 기본값과 최소/최대 범위.
- 뒤처졌을 때 재계산 상한(하루 목표가 과도해지는 것 방지 캡).
- "N절" 대상일 때 어떤 절을 뽑을지(코스 순서/사용자 선택).
- 프리셋 챌린지 초기 목록 큐레이션.
- 스트릭 UTC 자정 기준을 로컬 자정으로 바꿀지(기존 계약과의 정합성).
