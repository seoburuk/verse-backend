# 데일리루프 설계 (2026-07-22)

PIXBIBLE(Flutter 앱)에 **매일 재방문 습관을 만드는 리텐션 엔진**을 도입한다.
사용자가 의미 있는 암송 목표(플랜)를 정하면, 앱이 그것을 오늘치 목표로 쪼개주고,
매일 채우면 진행도가 눈에 띄게 차오르고 스트릭이 이어지며 **픽셀 마스코트가 반응한다**.
안 하면 마스코트가 배고파하며 저녁 알림으로 상기시킨다.

- **대상 플랫폼**: 앱(Flutter) 전용. 백엔드는 필요한 만큼만, 오프라인 우선(로컬 drift) 유지.
- **범위 상태**: 방향 확정 · 플랜 모델은 최소로 시작해 향후 발전(§9 열린 질문 참조).

---

## 1. 컨셉 한 줄

> 사용자가 "이 코스를 이번 달에 완주" 같은 **암송 플랜**을 정하면, 앱이 **오늘치 목표**로
> 쪼개주고("오늘 3절"), 매일 그걸 채우면 **진행바가 차오르고 스트릭이 이어지며 마스코트가 기뻐한다**.
> 안 하면 마스코트가 배고파하며 저녁에 알림으로 알려준다. 플랜을 다 외우면 **다음 플랜을 만들어 루프가 이어진다**.

핵심 통찰: 현재 스트릭은 "아무거나 1절"이라 **목적이 없다**. 사용자가 고른 목표를
일일 목표로 자동 분해하면, 매일 앱을 여는 이유가 "내가 정한 그 목표를 끝내기 위해"로 바뀐다 —
이것이 리텐션의 핵심 동력이다.

## 1-1. 듀오링고 플레이북 대비 (참고)

듀오링고식 코어(가벼운 XP 일일목표 + 스트릭 주인공 + 패스 자동 이어가기)와 비교해,
이 앱은 **"목표 플랜"을 중심**으로 간다(사용자 결정). 다만 듀오링고에서 가져오는 요소:

- **스트릭을 홈의 주인공으로** 크게 노출(기존 `StreakRepository` 재사용).
- **다음 절 자동 이어가기** = 듀오링고의 패스. 기존 `getResumeTarget`으로 이미 가능.
- **완료 축하 연출**(컨페티/격려 문구)로 재투자 감정 제공. 게임 재화 없음.
- 데일리 퀘스트·리그(랭킹 심화)는 향후(§9).

---

## 2. 접근 결정

- **A(사용자 지정 플랜 → 일일 목표 자동 산출)를 뼈대로** 한다. 프리셋 챌린지(C)는 향후.
- 보상은 **게임적 재화(카드/XP/빙결) 없음**. 진행도 갱신 + 마스코트 반응 + 축하 연출이 곧 보상이다.
- **마스코트(펫) 결속 메커니즘을 채택한다**. 오늘 목표를 채우면 마스코트가 배부르고/기뻐하고,
  며칠 안 하면 배고파한다(다마고치·듀오링고식). 마스코트는 **데일리루프 전용 동반자**이고,
  카드 수집(섹션 완료 수집물)은 별개로 기존대로 유지한다 — 두 시스템은 역할이 겹치지 않는다.
  - 상태는 **별도 재화·테이블 없이** 기존 스트릭/오늘치 완료 이력에서 파생한다(§7).
  - 톤 주의: 죄책감 과하지 않게 — "굶어 죽음"이 아니라 "시무룩/배고픔"까지만, 복귀하면 즉시 회복.

---

## 3. 완료 순간 · 스트릭 · 마스코트 (재투자 훅)

- 오늘 목표 절 수만큼 green 달성 → **오늘치 완료**. 그 순간:
  - 스트릭 기록(`recordActivityToday`), 홈의 오늘 목표 링이 꽉 참,
  - **마스코트가 "배부름/행복" 상태로 전환** + **축하 연출**(컨페티 + "오늘도 해냈어요!" 격려) 1회.
- 미완료 → 시간이 지나며 마스코트가 단계적으로 배고파함(표정 단계). 저녁에는 기존
  **스트릭 위험 알림**을 **"마스코트가 배고파해요 🍞"** 문구로 재활용해 상기시킨다.
- 마스코트 상태는 별도 재화 없이 스트릭/오늘치 완료 이력에서 파생(§7) — 진행도·스트릭·마스코트가 동기.

---

## 4. 홈 "오늘" 화면 (앱 시작 화면)

앱 시작 위치를 현재 `/courses`에서 **새 "오늘" 홈**으로 변경한다. 코스 탐색은 홈에서 진입하는
하위 화면으로 내린다.

레이아웃(위→아래):
1. 상단: **픽셀 마스코트(상태 반영)** + **스트릭 불꽃(크게)** + 목숨.
2. 중앙: **오늘 목표 링/바** — "오늘 3절 중 1절" 식으로 크고 눈에 띄게.
3. 그 아래: **플랜 전체 진행바** — "○○ 코스 6/23절 (이번 달 D-18)".
4. 메인 CTA: **"이어서 외우기"** → 플랜의 다음 미완료 절로 바로 진입.
5. 하단: 코스 탐색 / 대시보드 / 랭킹 진입.
6. 플랜이 없을 때: **"플랜 만들기"** 빈 상태 카드.

---

## 5. 플랜 모델 (최소 시작, 향후 발전)

- **플랜 = 대상 + 마감일**. **P1 대상 = 코스 하나**(코스의 모든 절이 플랜 범위).
  - 이유: 앱은 큐레이션된 코스/섹션의 `course_items`만 로컬에 갖고 있어, 임의의 "특정 장"은
    아직 콘텐츠가 없다. 코스는 `course_items`·진행도·resume에 그대로 매핑된다.
  - 마감일: 이번 주 / 이번 달 / 직접 날짜.
- **완주 시**: "🎉 완성!" → **다음 플랜(다른 코스) 만들기 화면**으로 유도해 루프 지속.
- **향후 발전(§9)**: 섹션 타깃, "특정 장" 타깃(백엔드 절 로딩 필요), "N절" 타깃, 프리셋 챌린지,
  동시 다중 플랜.

---

## 6. 오늘치 목표 자동 산출

- `남은 절 = 플랜 코스의 전체 절 − 완료(cleared) 절`.
- `남은 일수 = max(1, 마감일 − 오늘 + 1)` (오늘 포함, 마감 지나면 1로 클램프).
- `오늘 목표 = 남은 절 ≤ 0 ? 0 : ceil(남은 절 ÷ 남은 일수)`. 뒤처지면 자동 재계산해 따라잡기.
- **"오늘치 완료" 판정** = 오늘(UTC 일자) 새로 cleared된 플랜 코스 절 수 ≥ 오늘 목표.
- 완료 순간 스트릭 기록 + 축하 연출.
- **딕테이션 모드는 기존 규칙대로 진행도 미반영**(채점 계약 §7 유지).

---

## 7. 데이터 & 재사용

**신규 (로컬 drift 테이블) — 단 1개:**
- `memorization_plan` — 대상 코스 id · 마감일 · 상태(active/completed) · 생성 시각.

**파생 계산(테이블 없이):**
- 오늘 진행도·전체 진행도는 기존 `Progress`(cleared, updatedAt) + `CourseItems` 조인으로 계산.
  별도 `daily_progress` 테이블을 만들지 않는다(단순성 원칙).
- **마스코트 상태**(행복/보통/배고픔 단계)도 테이블 없이 파생한다:
  `StreakState.lastDay`(마지막 활동일)와 오늘치 완료 여부로 표정 단계를 계산 → `pet_state` 테이블 불필요.

**재사용:**
- `StreakRepository`(`lib/core/db/lives_streak_repository.dart`) — 완료 시 `recordActivityToday()`.
- `CourseRepository`의 다음 미완료 절 로직(`_nextUnclearedInCourse`) — "이어서 외우기"에 재사용.
- 알림 3종(`lib/core/notifications/reminder_service.dart`) — 그대로.
- 카드 수집(섹션 완료) — 손대지 않고 분리 유지.

**원칙**: 오프라인 우선. 전부 로컬 drift로 먼저 동작하고, 서버 동기화는 나중 단계.

---

## 8. 범위 / 단계 (Phasing)

- **P1** — `memorization_plan` 테이블 + 플랜 생성(코스+마감) + 오늘 홈 + 오늘목표/진행바 +
  "이어서 외우기" + 완주→다음 플랜 유도. 스트릭·알림·resume 재사용. (마스코트 없이도 완결 동작)
- **P2** — **마스코트 상태·애니메이션(파생 상태 구동)** + 스트릭 위험 알림 문구를 "마스코트 배고픔"으로
  교체 + 완료 축하 연출 강화. 오늘 홈을 앱 시작 화면으로 정식 승격 + 하단 내비 정리.
- **P3(향후 발전, §9 선결)** — 섹션/특정 장/N절 타깃, 프리셋 챌린지, 데일리 퀘스트, 서버 동기화.

---

## 9. 열린 질문 (향후 발전 시 확정)

- 플랜 대상 확장: 섹션 / "특정 장"(백엔드에서 임의 장 절 로딩 필요) / "N절".
- 마감일 기본값과 최소/최대 범위, 뒤처졌을 때 하루 목표 상한(캡).
- "오늘치" 판정의 날짜 기준: 스트릭은 UTC 자정, `Progress.updatedAt`은 로컬 시각 —
  정합성 정리 필요(§6은 UTC 기준으로 계산).
- 프리셋 챌린지 초기 목록 큐레이션.
- 동시 다중 플랜 허용 여부(P1은 활성 플랜 1개).
- 데일리 퀘스트·리그(랭킹 심화) 도입 여부.
- **마스코트 세부(P2 착수 전 확정)**: 표정 단계 수, 배고픔 진행 속도(며칠 만에 어느 단계?),
  회복 규칙(오늘치 완료만? 부분 진행 반영?), 마스코트 아트/애니메이션 에셋, 알림 문구 최종안.
