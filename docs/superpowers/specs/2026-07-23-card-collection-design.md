# 카드 컬렉션(도감) 설계 (2026-07-23)

성경 **동물**과 **인물(장비·상징물로 표현)** 카드를 모으는 도감. 관련 섹션을 다 외우면
카드가 해금되고, 섹션 완료 화면에서 뒤집어 공개한다. 하단 바 `카드` 탭의 플레이스홀더를 대체한다.

- **대상**: `verse-flutter/` 앱 전용(웹에도 컬렉션은 없다). 오프라인 우선.
- **선행**: 하단 바 `/cards` 탭(구현 완료, 현재 "준비 중" 플레이스홀더).
- **아트**: v1은 **아트 없이 플레이스홀더로 구현**한다. 로직을 먼저 완성하고 그림은 나중에 끼운다.
- **전제**: **카드 목록은 계속 업데이트된다.** 따라서 카탈로그를 코드가 아닌 데이터로 분리한다.

---

## 1. 컨셉 & 카드 표현 원칙

> 성경 동물과 인물을 모으는 도감. 관련 섹션을 외우면 한 장씩 채워진다.

- **동물**은 그대로 그린다: 양·사자·비둘기·낙타·물고기·독수리.
- **인물은 사람을 그리지 않고 그의 장비·상징물로 표현한다**:
  다윗→물맷돌, 모세→지팡이, 노아→방주, 골리앗→창, 엘리야→불수레.
- 이 원칙을 택한 이유:
  1. 예수·성인 묘사를 둘러싼 논란을 원천적으로 피한다.
  2. 저해상도 픽셀에서 **사람 얼굴은 뭉개지지만 물건은 식별된다** — 카드 간 실루엣 구분이 쉽다.
  3. 기존 마스코트 Shaun(양)의 픽셀 톤과 자연스럽게 이어진다.

---

## 2. 카탈로그는 데이터로 분리

카드 목록을 **번들 JSON**(`assets/cards/cards.json`)에 둔다. 기존 `assets/courses/courses.json`과
같은 패턴이라 구조가 낯설지 않고, **카드를 추가·수정할 때 Dart 코드를 건드리지 않는다**
(아이디어가 계속 업데이트될 것이므로 이 분리가 핵심이다).

카드 1장 스키마:

```json
{
  "id": "dove",
  "name": "비둘기",
  "name_en": "Dove",
  "kind": "animal",
  "description": "홍수가 끝났음을 알린 새.",
  "description_en": "The bird that announced the flood had ended.",
  "image": "dove.svg",
  "unlock": { "courseSlug": "genesis-walk", "sectionOrd": 3 }
}
```

- `kind`: `animal` | `figure`(장비로 표현되는 인물).
- `image`: `assets/cards/art/` 아래 파일명. **없거나 파일이 아직 없으면 플레이스홀더로 렌더**(§5).
- `unlock`: 해금 조건. 코스 슬러그 + 섹션 순번으로 가리킨다(§3).

---

## 3. 해금 조건 & 판정

**해금 단위 = 섹션 완료.** 카탈로그의 `unlock`이 가리키는 섹션의 모든 절을 외우면 그 카드가 해금된다.
섹션당 카드는 **0장 또는 1장**(매핑이 없는 섹션은 카드를 주지 않는다).

판정은 전부 **파생 계산**으로 한다 — 데일리루프·마스코트와 같은 철학:

| 필요한 것 | 방법 |
|---|---|
| 해금 여부 | 해당 섹션의 모든 `course_items`가 `cleared`인지 기존 `Progress` 조인으로 계산. **새 테이블 없음.** |
| 뒤집기를 이미 봤는가 | 축하 1회 가드와 동일하게 `SyncMeta`에 공개한 카드 id를 기록. 재방문 시 연출이 반복되지 않는다. |

---

## 4. 획득 연출 — 뒤집기

섹션 완료 화면(`SectionCompleteScreen`, 이미 happy 마스코트 + 축하가 있는 자리)에 붙인다.

- 그 섹션에 매핑된 카드가 있고 아직 공개하지 않았다면 **카드 뒷면**이 나타난다.
- 사용자가 **탭하면 뒤집히며 앞면 공개**.
- 확정 보상이라 "뭐가 나올지 모르는 재미"는 없지만, **내가 뒤집는 행위**가 기대감과 참여감을 만든다.
- 무작위 뽑기(가챠)는 하지 않는다 — 중복·사행성·확률 공개 의무를 모두 피한다.
- 카운터 상승·마스코트 전달 연출은 v1에 넣지 않는다(YAGNI). 필요하면 나중에.

---

## 5. 플레이스홀더 아트 전략

아트가 준비되기 전에도 기능이 완결되게 한다.

- `image` 파일이 없으면 **이름 첫 글자 + `kind`별 색 타일**로 대체 렌더한다.
- **잠긴 카드**는 같은 타일을 검은 실루엣으로 렌더한다(아트가 생기면 그 이미지의 실루엣).
- 아트가 준비되면 `assets/cards/art/`에 파일을 넣고 `cards.json`의 `image`만 채우면
  **코드 수정 없이 교체**된다.

---

## 6. 도감 갤러리 화면 (`/cards` 탭)

현재 "준비 중" 플레이스홀더(`CardCollectionScreen`)를 대체한다.

- **3열 그리드**. 해금된 카드는 그림 + 이름, **잠긴 카드는 실루엣**으로 표시한다 —
  존재는 보이되 정체는 가리는 것이 "모으고 싶다"를 만드는 장치다.
- 상단에 **수집 카운터**("23/40").
- 카드를 탭하면 **상세**: 그림 · 이름 · 설명.
- 잠긴 카드의 상세는 **해금 힌트**를 보여준다(예: "창세기 걷기 3번째 섹션").

---

## 7. 데이터 & 재사용

**신규 drift 테이블: 없음.**

| 필요한 것 | 출처 |
|---|---|
| 카드 카탈로그 | `assets/cards/cards.json` (신규 번들 에셋) |
| 섹션 완료 여부 | 기존 `CourseItems` + `Progress` 조인 |
| 공개(뒤집기) 기록 | 기존 `SyncMeta` (via `AppSettingsRepository`) |
| 카드 탭 자리 | 기존 `/cards` 라우트 + `CardCollectionScreen` |
| 획득 연출 자리 | 기존 `SectionCompleteScreen` |

---

## 8. 테스트

- **카탈로그 파싱**: `cards.json`이 스키마대로 로드되고, 잘못된 항목이 있어도 앱이 죽지 않는다.
- **해금 판정 순수 함수**: 섹션 전부 cleared → 해금, 일부만 → 잠김, 매핑 없는 섹션 → 카드 없음.
- **공개 1회 가드**: 같은 섹션을 다시 완료 화면으로 들어가도 뒤집기가 다시 뜨지 않는다.
- **갤러리 위젯**: 해금/잠김이 각각 이름/실루엣으로 렌더되고 카운터가 맞는다.
- **플레이스홀더**: `image`가 없는 카드도 크래시 없이 타일로 렌더된다.

---

## 9. 범위 밖

- **랜덤 뽑기(가챠)·재화·중복 교환** — 채택하지 않는다.
- **카드 공유 이미지** — 기존 `ShareCard`(구절/마일스톤/스트릭)와 별개다. 필요하면 후속.
- **서버 동기화·카탈로그 원격 갱신** — v1은 번들 JSON만. `courses.json`처럼 버전 체크로
  원격 갱신하는 것은 후속 과제.
- **실제 카드 아트 제작** — 이 스펙은 플레이스홀더까지만 책임진다.
- **카드 목록 확정** — 카드 아이디어는 계속 업데이트되므로, 이 스펙은 **스키마와 메커니즘**을
  정의하고 구체적 카드 목록은 데이터로 남긴다.
