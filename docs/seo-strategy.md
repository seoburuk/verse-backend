# SEO 전략 — PIX BIBLE

작성일: 2026-07-03

## 현재 상태

- `layout.tsx`에 기본 `metadata`(title/description/OG) 존재
- `robots.ts`, `sitemap.ts` 존재하지만 최소 수준
  - `sitemap.ts`: 홈(`/`) 1개만 등록, `metadataBase` 미설정으로 상대경로 사용 중 (절대 URL 필요)
  - `robots.ts`: `/courses`를 차단 중 — 코스 목록/상세가 앱의 핵심 콘텐츠인데 검색 노출에서 제외되고 있음
- OG 이미지, 페이지별 키워드, JSON-LD 구조화 데이터, 네이버/구글 소유 확인 태그 없음
- 콘텐츠 대부분이 로그인 뒤에 있어 크롤러가 색인할 공개 텍스트가 거의 없음 (가장 큰 병목)

## 전략 (우선순위 순)

### 1. 크롤러가 볼 수 있는 공개 콘텐츠 확보 — 핵심
- 코스 목록·코스 상세 페이지를 비로그인 공개로 전환 (읽기 전용, 암송 시작 시에만 로그인 유도)
  - 30개 워밍업 코스 각각이 "사랑에 관한 성경구절" 등 검색어 랜딩 페이지 역할
- 절(verse) 단위 공개 페이지 추가 (예: `/verse/john-3-16`) — KJV 원문 + 한글 설명
  - 롱테일 검색 트래픽 확보, 66권 × 장 단위로도 수천 페이지 규모의 색인 자산 가능
- 서버 컴포넌트로 렌더링해 HTML에 텍스트가 실제로 담기도록 함 (네이버 크롤러는 JS 렌더링이 약해 서버 렌더링 필수)

### 2. 기술 SEO 정비 (코드 작업)
- `metadataBase` 설정 + `sitemap.ts`를 절대 URL로 수정, 공개 코스/절 페이지를 동적으로 sitemap에 포함
- `robots.ts`에서 `/courses` 차단 해제 (로그인·대시보드·설정만 차단 유지)
- 페이지별 `generateMetadata`로 고유 title/description 부여
  (예: "「코스명」 — 주제별 성경구절 암송 | PIX BIBLE")
- OG 이미지: `opengraph-image.tsx`로 픽셀 스타일 동적 생성
- JSON-LD 구조화 데이터: `WebSite`, `WebApplication`, 절 페이지에 `Article`/`Quotation`

### 3. 네이버 특화
- 네이버 서치어드바이저 등록 → 소유 확인 메타태그 추가 → sitemap 제출 (자발적 크롤링이 약해 미등록 시 색인 거의 안 됨)
- 네이버 검색 상단은 블로그/카페 콘텐츠가 우선하는 경향 → 네이버 블로그 병행 운영("성경 암송 방법" 등) 후 앱 링크 유도가 웹 SEO보다 효과적일 수 있음
- 메타 description은 한글 위주, 핵심을 80자 내외 앞부분에 배치

### 4. 구글 특화
- Google Search Console 등록 + sitemap 제출
- Core Web Vitals: Next.js 기본기 + 이미지/폰트(Galmuri) 최적화
- 타겟 키워드: "성경 암송 앱", "KJV 암송", "성경구절 외우기", "주제별 성경구절" — 경쟁 낮은 니치, 콘텐츠 페이지 확보 시 상위 노출 가능성 높음

### 5. 측정
- Search Console + 네이버 서치어드바이저에서 색인 수·노출 키워드를 2~4주 주기로 확인
- 유입 키워드에 맞춰 절 페이지 증설

## 실행 순서 제안

1. 코드 작업 (반나절): `metadataBase`·sitemap·robots 수정, 페이지별 메타데이터, OG 이미지, JSON-LD, 네이버/구글 인증 태그 자리 마련
2. 공개 페이지 전환 (하루): 코스 목록·상세 비로그인 열람 (제품 결정 필요)
3. 콘텐츠 확장 (지속): 절 단위 공개 페이지
4. 외부 등록 (30분): 네이버 서치어드바이저·Google Search Console 등록 및 sitemap 제출

## 미결정 사항

- 배포 도메인 (`metadataBase` 값 확정에 필요)
- 코스 목록/상세를 비로그인 공개로 전환할지 여부 — 제품 정책 결정 필요
